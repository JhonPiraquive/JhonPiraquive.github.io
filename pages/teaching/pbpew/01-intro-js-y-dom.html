¿Qué es JavaScript?

JavaScript es un lenguaje de programación diseñado al principio para dar comportamiento a las páginas web en el navegador: validar formularios, reaccionar a clics, cambiar el contenido sin recargar toda la página, etc.

Hoy también se usa fuera del navegador (servidores con Node.js, herramientas de desarrollo, scripts). El estándar que define el lenguaje se publica como ECMAScript; cuando leas “ES2015”, “ES6” o “ES2024”, se refieren a versiones de ese estándar.

Características principales

  • Interpretado: el motor (navegador u otro entorno) lee y ejecuta el código sin un paso de compilación obligatorio como en algunos lenguajes clásicos.
  • Dinámico: el tipo de una variable puede cambiar según el valor que guardes (tipado dinámico).
  • Multiparadigma: puedes programar con funciones, objetos y, desde ES6, también con class para orientación a objetos.
  • Event-driven en el navegador: mucho código “reacciona” a eventos (clic, tecla, carga de página).
  • Un solo hilo en el modelo clásico del navegador para tu página; la asincronía (más adelante en el curso) organiza tareas sin bloquear la interfaz.

¿Para qué se usa JavaScript?

Desarrollo web (front-end)

JS manipula el DOM, valida datos, comunica con APIs y construye experiencias interactivas. Muchos sitios combinan HTML (estructura), CSS (presentación) y JavaScript (comportamiento).

Aplicaciones en servidor (Node.js)

Node.js es un entorno que ejecuta JavaScript fuera del navegador. Sirve para APIs REST, herramientas de línea de comandos, automatización y servidores en tiempo real.

Aplicaciones móviles (TypeScript / JSX → React Native)

En ecosistemas como React Native sueles escribir en TypeScript o usar sintaxis JSX (HTML dentro de JavaScript). Un transpilador (por ejemplo Babel) convierte ese código a JavaScript estándar. El framework ejecuta la lógica y, mediante su puente nativo, traduce la interfaz y las llamadas a componentes que en Android terminan interactuando con Kotlin/Java y en iOS con Swift/Objective-C, produciendo el ejecutable de la app.

Videojuegos

Existen motores y bibliotecas (Phaser, Three.js, engines web) que usan JavaScript o WebGL para juegos en navegador o empaquetados como aplicación.

Historia breve y vídeo

JavaScript nació en 1995 en Netscape, de la mano de Brendan Eich, con el nombre inicial “Mocha” y luego “LiveScript”. Se estandarizó bajo el nombre ECMAScript. La historia incluye la “guerra de los navegadores”, la evolución por versiones ES y el auge de Node.js y los frameworks modernos.

¿Qué es el DOM?

Cuando el navegador lee un documento HTML, construye una representación en memoria: el Document Object Model (DOM). Es un árbol de nodos: documento, elementos (<p>, <div>), texto, atributos, etc.

JavaScript puede recorrer y modificar ese árbol: crear nodos, cambiar texto, estilos o escuchar eventos. No confundas el DOM con el código fuente HTML en disco: el DOM es la estructura viva en memoria que puede cambiar sin volver a pedir el archivo al servidor.

Resumen

  • JavaScript da comportamiento a la web y se ha extendido a servidor y otras plataformas.
  • El DOM es el árbol de objetos que representa la página; JS lo manipula en tiempo de ejecución.
  • En móvil con React Native, el código TS/JSX suele transpilarse a JS y el framework habla con capas nativas.
