Tema 02

JavaScript en HTML, “Hola mundo” y consola

Dónde colocar scripts, archivos externos, comentarios y el uso de console para depurar.

Leer

Objetivos

  • Diferenciar JavaScript inline y externo.
  • Saber colocar <script> en HTML sin romper el parseo del DOM.
  • Escribir “Hola mundo” y usar los métodos principales de console.
  • Usar comentarios para documentar el código.

JavaScript inline y externo

Inline: código dentro del mismo HTML, en un atributo (onclick, etc.) o entre etiquetas <script>.

Externo: código en un archivo .js enlazado con <script src="..."></script>. Mejor para reutilizar y cachear.

¿Dónde poner el <script>?

  • Al final del <body> (recomendado en páginas clásicas): cuando el script se ejecuta, el DOM ya está construido.
  • En el <head> con defer: el navegador descarga el script en paralelo y lo ejecuta tras parsear el HTML.
<!doctype html>
<html lang="es">
<head>
  <meta charset="utf-8" />
  <title>Ejemplo</title>
  <script src="app.js" defer></script>
</head>
<body>
  <p id="msg">Hola</p>
  <script>
    console.log("Script al final del body");
  </script>
</body>
</html>

Hola mundo con JavaScript

<script>
  console.log("Hola mundo");
</script>

Si quieres modificar la página al cargar:

document.body.innerHTML = "<p>Hola mundo desde JS</p>";

Consola del navegador y tipos de mensajes

La consola muestra salida y errores. Métodos habituales:

  • console.log: mensaje general.
  • console.info: informativo (apariencia puede variar según el navegador).
  • console.warn: advertencias en amarillo.
  • console.error: errores en rojo, con traza.
  • console.table: tablas a partir de arrays u objetos.
  • console.time / console.timeEnd: medir duración.
console.log("Paso 1");
console.warn("Cuidado: valor límite");
console.error("Algo falló");
console.table([
  { nombre: "Ana", nota: 5 },
  { nombre: "Luis", nota: 4.5 },
]);
console.time("bucle");
for (let i = 0; i < 1e6; i++) {}
console.timeEnd("bucle");

Comentarios en el código

// Comentario de una línea

/*
  Comentario de
  varias líneas
*/
const pi = 3.14; // explicar constantes poco obvias

Buena práctica: comenta el “por qué”, no lo obvio. El código claro reduce la necesidad de comentarios largos.

Resumen

  • Prefiere JS externo + defer o script al final del body.
  • La consola es tu aliada con log, warn, error y table.
  • Los comentarios ayudan a futuros lectores (incluido tú dentro de un mes).
