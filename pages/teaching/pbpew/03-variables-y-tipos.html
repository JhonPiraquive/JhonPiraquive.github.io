Tema 03

Variables, hoisting y tipos de datos

var, let, const; cómo el motor “eleva” declaraciones; strings, números, booleanos, undefined y null.

Leer

Objetivos

  • Declarar variables con let y const, y conocer el papel legacy de var.
  • Explicar el hoisting en términos sencillos.
  • Identificar tipos primitivos y reconocerlos en código.

¿Qué es una variable?

Una variable es un nombre que el programa asocia a un valor en memoria. Sirve para guardar datos que cambian (contadores, texto del usuario) o constantes de configuración.

var, let y const

  • let: variable que puedes reasignar. Alcance de bloque ({}).
  • const: enlace constante al mismo valor; no puedes reasignar la variable (si el valor es un objeto, sí puedes mutar sus propiedades).
  • var: estilo antiguo; alcance de función; puede sorprender dentro de bucles. En código nuevo prefiere let/const.
let contador = 0;
contador = contador + 1;

const URL_API = "https://api.ejemplo.com";
// URL_API = "otro"; // Error: reasignación prohibida

const usuario = { nombre: "Ana" };
usuario.nombre = "Luis"; // válido: mutación del objeto

var x = 1;
if (true) {
  var x = 2; // misma variable x "global" de función
}
console.log(x); // 2 con var — con let sería distinto

Hoisting (“elevación”)

Antes de ejecutar tu código, el motor procesa las declaraciones. Las declaraciones var se “elevan” al inicio de su función y quedan inicializadas como undefined. Con let y const también existe hoisting, pero entran en la “zona muerta temporal”: no puedes leerlas antes de la línea de declaración.

console.log(a); // undefined (var declarado más abajo)
var a = 5;

try {
  console.log(b);
} catch (e) {
  console.error("Error al acceder a b antes de declarar:", e.name);
}
let b = 10;

Tipos de datos principales

  • string: texto. Ejemplo real: el nombre del usuario. "hola", 'mundo', template strings en el tema 11.
  • number: números enteros y decimales (IEEE 754). Ejemplo: precio, edad, coordenada.
  • boolean: true o false. Ejemplo: “¿está aceptado el término?”.
  • undefined: valor por defecto si no asignaste nada; también si una función no devuelve valor.
  • null: “vacío intencional”; suele indicar “sin objeto” por decisión del programador.
const nombre = "María"; // string
const edad = 22; // number
const aprobado = true; // boolean
let nota; // undefined hasta que asignas
const sesion = null; // sin usuario activo

typeof nombre; // "string"
typeof edad; // "number"
typeof aprobado; // "boolean"
typeof nota; // "undefined"
typeof sesion; // "object" (peculiaridad histórica de typeof con null)

Resumen

  • Usa const por defecto, let cuando necesites reasignar, evita var en código nuevo.
  • Hoisting: declaraciones movidas en la fase de compilación; let/const no se pueden leer antes de su línea.
  • Conoce typeof para inspeccionar tipos en tiempo de ejecución.
