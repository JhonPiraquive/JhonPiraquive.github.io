Sincronía vs asincronía · temporizadores · promesas · async/await

No bloquear la interfaz; programar tareas que terminan “más tarde”. Template literals.

Leer

¿Qué cambia con lo asíncrono?

Código sincrónico se ejecuta línea a línea; cada línea espera a la anterior. Código asíncrono programa trabajo que completa más tarde (red, temporizador); el navegador puede seguir respondiendo a la UI.

setTimeout y setInterval

const id = setTimeout(() => {
  console.log("Pasó 1 segundo");
}, 1000);

const tick = setInterval(() => {
  console.log("tic");
}, 2000);
clearInterval(tick);

Promesas

Una Promise representa un valor futuro: pendiente, cumplida (resolved) o rechazada (rejected).

function esperar(ms) {
  return new Promise((resolve) => setTimeout(resolve, ms));
}

esperar(500).then(() => console.log("listo"));

then, catch, finally

fetch("/api/datos.json")
  .then((r) => r.json())
  .then((data) => console.log(data))
  .catch((err) => console.error("Fallo", err))
  .finally(() => console.log("Fin del intento"));

async / await

async function cargar() {
  try {
    const res = await fetch("/api/datos.json");
    const data = await res.json();
    console.log(data);
  } catch (e) {
    console.error(e);
  }
}

Template literals

const nombre = "Diana";
const saludo = `Hola ${nombre}, bienvenida a PBPEW`;
const multilinea = `
  Línea 1
  Línea 2`;

Resumen

  • Temporizadores y red son asíncronos; encadenas con promesas o async/await.
  • catch maneja errores en promesas; finally limpia siempre.
  • Los template literals interpolan expresiones con ${}.
