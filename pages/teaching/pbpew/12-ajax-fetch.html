AJAX y la API fetch

Pedir datos al servidor sin recargar la página completa; comparación breve con XMLHttpRequest.

Leer

¿Qué es AJAX?

AJAX (Asynchronous JavaScript And XML) es un enfoque: usar JavaScript para enviar peticiones HTTP en segundo plano y actualizar parte de la página con la respuesta. Hoy las respuestas suelen ser JSON, no solo XML.

XMLHttpRequest (legado)

API antigua pero aún presente; más verbosa. Ejemplo mínimo conceptual:

const xhr = new XMLHttpRequest();
xhr.open("GET", "https://jsonplaceholder.typicode.com/todos/1");
xhr.onload = function () {
  if (xhr.status >= 200 && xhr.status < 300) {
    console.log(JSON.parse(xhr.responseText));
  }
};
xhr.send();

fetch moderno

fetch devuelve una Promise. Recuerda comprobar response.ok y parsear JSON si aplica.

async function obtenerTodo() {
  const res = await fetch("https://jsonplaceholder.typicode.com/todos/1");
  if (!res.ok) throw new Error("HTTP " + res.status);
  const data = await res.json();
  console.log(data.title);
}

obtenerTodo().catch(console.error);

CORS: desde una página en el navegador solo puedes llamar a otros orígenes si el servidor lo permite. Si abres el archivo como file://, algunas demos de red pueden fallar; usa un servidor estático local si hace falta.

  • AJAX describió la revolución de las SPA; hoy usamos principalmente fetch o bibliotecas.
